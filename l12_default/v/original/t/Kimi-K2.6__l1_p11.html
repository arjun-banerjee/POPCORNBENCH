We need to write a custom CUDA kernel for the operation C[b,i,j,k] = sum_l A[b,i,j,l] * B[l,k]. This is essentially a batched matrix multiplication where A has shape [b, i, j, l] and B has shape [l, k], and the output is [b, i, j, k]. It's equivalent to reshaping A to [b*i*j, l] and doing a matrix multiplication with B [l, k], resulting in [b*i*j, k], then reshaping back to [b, i, j, k]. PyTorch's einsum likely does exactly this under the hood using highly optimized BLAS (e.g., cuBLAS).
