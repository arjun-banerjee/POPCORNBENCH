The inputs: predictions shape (batch_size, num_classes), targets shape (batch_size,). predictions are logits (pre-softmax). cross_entropy reduces by mean by default? torch.nn.functional.cross_entropy default reduction='mean', so output is scalar float. We need to write a custom kernel that computes cross entropy loss efficiently. We have batch_size=32768, num_classes=4096. So total elements = 134,217,728 (~134M). Memory bandwidth bound: reading predictions (fp32) 134M * 4 bytes = 536 MB. Reading targets (int64) 32768*8=256KB. Writing output scalar negligible. Compute: we need per row max, subtract, exp, sum, log, then compute loss = (log(sum(exp(x - max))) + (-x_target + max)) averaged over batch. We can implement a fused kernel that reads each row, computes max, then second pass (or single pass if we store in shared memory or registers). Since row size 4096, we can have each threadblock process one or more rows. With 4096 elements per row, using reduction in shared memory. Given large batch size (32768), many rows. We can have each block handle a single row: blockDim.x = 1024 or 512. For 4096 elements, 1024 threads can do parallel reduction. But we want to maximize occupancy and memory bandwidth. Each row is independent, so launch 32768 blocks. Each block has e.g. 1024 threads doing 4 loads each. That is a lot of threads: 33.5M threads - may be too many? But we can use 256 threads per block, each handling 16 elements (4096/256=16). That gives 32768 blocks * 256 = 8,388,608 threads. Occupancy decent.
