import torch
import torch.nn as nn

class Model(nn.Module):
    """
    Selective scan (Mamba's S6 layer): projects input to derive input-dependent SSM parameters
    (delta, B, C), discretizes a learned state matrix A, runs the linear recurrence, and
    applies a skip connection via D. The fusion opportunity is keeping all intermediates
    (discretized parameters, hidden states) in SRAM rather than materializing them to global memory.

    Args:
        d_model (int): Input feature dimension.
        d_state (int): Hidden state dimension per channel (N in Mamba, typically 16).
    """
    def __init__(self, d_model: int, d_state: int):
        super(Model, self).__init__()
        self.d_model = d_model
        self.d_state = d_state
        self.A_log = nn.Parameter(torch.randn(d_model, d_state))
        self.proj_B = nn.Linear(d_model, d_state, bias=False)
        self.proj_C = nn.Linear(d_model, d_state, bias=False)
        self.proj_delta = nn.Linear(d_model, d_model, bias=True)
        self.D = nn.Parameter(torch.ones(d_model))

    def forward(self, x: torch.Tensor) -> torch.Tensor:
        B, L, D = x.shape
        N = self.d_state

        delta = torch.nn.functional.softplus(self.proj_delta(x))
        B_t = self.proj_B(x)
        C_t = self.proj_C(x)

        A = -torch.exp(self.A_log)
        dA = torch.exp(delta.unsqueeze(-1) * A.unsqueeze(0).unsqueeze(0))
        dB = delta.unsqueeze(-1) * B_t.unsqueeze(2)
        x_db = x.unsqueeze(-1) * dB

        h = torch.zeros(B, D, N, device=x.device, dtype=x.dtype)
        outputs = []
        for t in range(L):
            h = dA[:, t] * h + x_db[:, t]
            y_t = (h * C_t[:, t].unsqueeze(1)).sum(dim=-1)
            outputs.append(y_t)

        y = torch.stack(outputs, dim=1)
        y = y + x * self.D.unsqueeze(0).unsqueeze(0)
        return y

batch_size = 16

A = -exp(A_log) -> (D, N) 5. dA = exp(delta.unsqueeze(-1) * A.unsqueeze(0).unsqueeze(0)) -> (B, L, D, N) = exp(delta[:,:,:,None] * A[None,None,:,:]) 6. dB = delta.unsqueeze(-1) * B_t.unsqueeze(2) -> (B, L, D, N) = delta[:,:,:,None] * B_t[:,:,None,:] 7. x_db = x.unsqueeze(-1) * dB -> (B, L, D, N) = x[:,:,:,None] * dB 8. Initialize h = zeros(B, D, N) 9. Then sum over N -> (B, D). 10. y = stack(outputs, dim=1) -> (B, L, D) 11. y = y + x * D.unsqueeze(0).unsqueeze(0) -> (B, L, D) So overall it's a selective scan. We need to write a CUDA kernel that does all this fused, specifically the recurrence and output computation, and also the linear projections + softplus. We can use `torch.utils.cpp_extension.load_inline` to compile. Given the problem statement, the batch size is 16, d_model=256, d_state=16, seq_len=2048. That's B=16, L=2048, D=256, N=16. Total elements in y: 16*2048*256 = 8,388,608. In fp32 that's ~33.5 MB. dA, dB, x_db are huge if materialized: B*L*D*N = 16*2048*256*16 = 134,217,728 elements -> 512 MB each. So materializing them is expensive. The kernel should compute them on the fly. We need to think about how to parallelize. The recurrence is sequential over time t. But each (B, D, N) element is independent? Let's see: h_{t+1}[b,d,n] = dA[b,t,d,n] * h_t[b,d,n] + x_db[b,t,d,n] where dA = exp(delta[b,t,d] * A[d,n]) and x_db = x[b,t,d] * delta[b,t,d] * B_t[b,t,n] Wait: dB = delta * B_t. So x_db = x * dB = x * delta * B_t. So x_db[b,t,d,n] = x[b,t,d] * delta[b,t,d] * B_t[b,t,n]. Note: delta depends on d, B_t depends on n. x depends on d. So x_db = x[b,t,d] * delta[b,t,d] * B_t[b,t,n]. Then y[b,t,d] = sum_n h_t[b,d,n] * C_t[b,t,n]. Wait: after updating h at time t, we compute y_t using h. In the code: h = dA[:,t] * h + x_db[:,t] y_t = (h * C_t[:,t].unsqueeze(1)).sum(dim=-1) So the recurrence uses previous h, then computes output. For a given (b,d,n), the recurrence is: h_{t}[b,d,n] = exp(delta[b,t,d] * A[d,n]) * h_{t-1}[b,d,n] + x[b,t,d] * delta[b,t,d] * B_t[b,t,n] And the contribution to y[b,t,d] from n is h_t[b,d,n] * C_t[b,t,n]. So for each (b,d), we could compute a vector of size N=16 sequentially over t. There are B*D = 16*256 = 4096 such independent recurrences. Each recurrence length L=2048, dimension N=16. This is a perfect setup for CUDA: assign one thread block per (b,d) or one warp per (b,d), etc. But we also need to compute the linear projections for B_t, C_t, delta, etc. We have three linear projections: proj_delta (D->D, with bias), proj_B (D->N), proj_C (D->N). Input x is (B, L, D). We could precompute these projections using cuBLAS (by calling torch.matmul). Or we can compute them inside the kernel if we load x[b,t,d] and multiply by weights. But loading the weight matrices (D*D for delta, D*N for B and C) from global memory inside each thread might be expensive. Given D=256, N=16, L=2048, B=16. Computing proj_delta: (B*L, D) @ (D, D) -> 16*2048*256*256 = ~2.1B mul-adds. That's a lot of compute. We could use a fused kernel for that? Or rely on PyTorch's efficient linear implementation. However, the reference implementation uses `nn.Linear`, which PyTorch handles efficiently (cuBLAS). So if we just replace the whole forward with a custom kernel, we might not beat PyTorch's linear + elementwise + loop unless we fuse the linear ops into the scan. But wait: the reference implementation uses a Python loop over t, which is very slow. Even if we just use PyTorch's ops for the linear parts and a custom CUDA kernel for the scan, we could probably beat it easily. But to maximize speedup, fusing everything is better.
