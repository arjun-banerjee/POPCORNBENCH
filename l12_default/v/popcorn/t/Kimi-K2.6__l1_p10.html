import torch
import torch.nn as nn


class Model(nn.Module):
    """
    Edge-only sampled dense-dense matmul.

    For each graph edge (src, dst), compute a dot product between source and
    destination node embeddings. This pattern appears in link prediction,
    sampled attention, and graph contrastive learning.
    """

    def __init__(self):
        super().__init__()

    def forward(
        self,
        src_idx: torch.Tensor,
        dst_idx: torch.Tensor,
        src_feat: torch.Tensor,
        dst_feat: torch.Tensor,
    ) -> torch.Tensor:
        src = src_feat[src_idx.long()]
        dst = dst_feat[dst_idx.long()]
        return (src * dst).sum(dim=-1)

Compute: gather src_feat[src_idx] and dst_feat[dst_idx], multiply elementwise, sum over feat_dim. Output shape (num_edges,). This is essentially embedding lookup + elementwise multiply reduction. We need to write custom CUDA kernel, compile with load_inline, and call from ModelNew.forward. Need to be faster than PyTorch reference. PyTorch reference does indexing (src = src_feat[src_idx.long()], dst = dst_feat[dst_idx.long()], then sum over last dim). The indexing will create two tensors of shape (num_edges, feat_dim), memory bound. For 16384 edges and 128 dims, that's 2 * 16384 * 128 * 4 bytes = ~16.7 MB. Summation also memory bound.
