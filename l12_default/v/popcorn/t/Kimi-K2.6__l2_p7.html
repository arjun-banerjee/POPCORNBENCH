import torch
import torch.nn as nn

class Model(nn.Module):
    """
    Performs a depthwise-separable 1D convolution: a depthwise Conv1d (each channel independently)
    followed by a pointwise Conv1d (1x1, mixes channels). Used in Enformer and genomic sequence
    models for efficient spatial+channel factorized processing.

    Args:
        in_channels (int): Number of channels in the input.
        out_channels (int): Number of channels produced by the pointwise convolution.
        kernel_size (int): Kernel size of the depthwise convolution.
        stride (int, optional): Stride of the depthwise convolution. Defaults to 1.
        padding (int, optional): Padding for the depthwise convolution. Defaults to 0.
        bias (bool, optional): If True, adds a learnable bias. Defaults to False.
    """
    def __init__(self, in_channels: int, out_channels: int, kernel_size: int, stride: int = 1, padding: int = 0, bias: bool = False):
        super(Model, self).__init__()
        self.depthwise = nn.Conv1d(in_channels, in_channels, kernel_size, stride=stride, padding=padding, groups=in_channels, bias=bias)
        self.pointwise = nn.Conv1d(in_channels, out_channels, kernel_size=1, bias=bias)

    def forward(self, x: torch.Tensor) -> torch.Tensor:
        x = self.depthwise(x)
        x = self.pointwise(x)
        return x

reasoning
We need to write a custom CUDA kernel for a depthwise separable 1D convolution. The reference implementation uses two `nn.Conv1d` layers: depthwise (groups=in_channels) and pointwise (1x1). The goal is to fuse these operations into a single kernel to reduce memory traffic and improve performance. Key parameters: - batch_size = 32 - in_channels = 256 - out_channels = 512 - kernel_size = 9 (depthwise kernel size) - seq_len = 4096 - stride = 1 - padding = 4 - bias = False (from get_init_inputs, but class also accepts bias) The kernel should handle arbitrary batch_size, in_channels, out_channels, kernel_size, seq_len, stride, padding, and bias presence. But we can optimize for the known dimensions while keeping generic support. The operation: 1. Input: x of shape [batch_size, in_channels, seq_len] 2. Depthwise conv: groups=in_channels, kernel_size=K, stride=S, padding=P, in_channels=out_channels for this layer = in_channels. Output size L_out = floor((seq_len + 2*P - K) / S) + 1. With seq_len=4096, K=9, P=4, S=1 => L_out = (4096 + 8 - 9)/1 + 1 = 4096. 3. Pointwise conv: kernel_size=1, stride=1, padding=0, in_channels=in_channels, out_channels=out_channels. Output size = L_out. So overall: y[b, o, t] = sum_{c=0}^{in_channels-1} ( sum_{k=0}^{K-1} x[b, c, t*S + k - P] * w_dw[c, k] ) * w_pw[o, c] + bias_pw[o] (+ maybe bias_dw? Wait, reference has bias on both convs if bias=True.) But if bias=True, depthwise adds bias per channel, pointwise adds bias per output channel. So we need to incorporate both biases. Actually we can fuse: after depthwise, we have intermediate z[b,c,t] = sum_k x[...] * w_dw[c,k] + bias_dw[c]. Then pointwise: y[b,o,t] = sum_c z[b,c,t] * w_pw[o,c,0] + bias_pw[o]. If bias=False (default implied by get_init_inputs), then both biases are zero. Fused expression: y[b, o, t] = sum_{c=0}^{C_in-1} w_pw[o, c] * ( sum_{k=0}^{K-1} x[b, c, t*S + k - P] * w_dw[c, k] + bias_dw[c] ) + bias_pw[o] We can expand: = sum_c sum_k x[b,c,t*S+k-P] * w_pw[o,c] * w_dw[c,k] + sum_c w_pw[o,c] * bias_dw[c] + bias_pw[o] If bias_dw exists, we can precompute `bias_fused[o] = sum_c w_pw[o,c] * bias_dw[c] + bias_pw[o]` during initialization or inside the kernel. Since weights are fixed after initialization but updated during training, we can't precompute on CPU at runtime each forward without overhead? But we can precompute a fused bias buffer in `ModelNew.__init__` or inside forward on CPU (cheap) and pass it to kernel. Alternatively, compute inside the kernel: after accumulating depthwise result for a channel, add bias_dw (if any), multiply by w_pw, accumulate across channels, then add bias_pw. Memory bound or compute bound? Let's calculate FLOPs. Depthwise: batch * in_channels * L_out * K = 32 * 256 * 4096 * 9 ≈ 302M FLOPs. Pointwise: batch * out_channels * in_channels * L_out * 1 = 32 * 512 * 256 * 4096 ≈ 17.2G FLOPs. Total ≈ 17.5G FLOPs. FP32 performance of GPU: maybe 20-100 TFLOPS. So compute time at peak is ~0.2-0.9 ms. Memory traffic for reference: read input (32*256*4096*4 = 134MB), write intermediate (32*256*4096*4 = 134MB), read dw weights (256*9*4=9KB), read pw weights (512*256*4=524KB), write output (32*512*4096*4 = 268MB). Total memory ~536MB plus intermediate reads/writes. At 1 TB/s memory bandwidth, that's ~0.5ms. So there's room to speed up by fusing.
