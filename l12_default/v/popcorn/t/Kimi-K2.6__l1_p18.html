import torch
import torch.nn as nn


class Model(nn.Module):
    """
    Multi-head sparse aggregation with per-edge attention weights.

    Each edge carries one weight per head, and source node features are arranged
    as `(num_nodes, num_heads, head_dim)`. The output aggregates weighted
    source features into `(num_nodes, num_heads, head_dim)`.
    """

    def __init__(self):
        super().__init__()

    def forward(
        self,
        row_ptr: torch.Tensor,
        col_idx: torch.Tensor,
        edge_weight: torch.Tensor,
        node_feat: torch.Tensor,
    ) -> torch.Tensor:
        num_nodes = row_ptr.numel() - 1
        num_heads = node_feat.shape[1]
        head_dim = node_feat.shape[2]
        out = torch.zeros(num_nodes, num_heads, head_dim, dtype=node_feat.dtype, device=node_feat.device)

        for dst in range(num_nodes):
            start = int(row_ptr[dst].item())
            end = int(row_ptr[dst + 1].item())
            if end > start:
                src = col_idx[start:end].long()
                weights = edge_weight[start:end].unsqueeze(-1)
                out[dst] = (node_feat[src] * weights).sum(dim=0)

        return out


num_nodes = 320
avg_degree = 14
num_heads = 8

reasoning
We need to write a custom CUDA kernel for this sparse aggregation. The operation: given a CSR-like representation (`row_ptr`, `col_idx`), edge weights of shape `(num_edges, num_heads)`, and node features of shape `(num_nodes, num_heads, head_dim)`, we want to compute `out` of shape `(num_nodes, num_heads, head_dim)` where for each destination node `dst`, for each incident edge `e` in range `[row_ptr[dst], row_ptr[dst+1])`, `src = col_idx[e]`, and `out[dst] += edge_weight[e, :] * node_feat[src, :, :]`. This is similar to a sparse matrix multiplication where the sparse matrix is `(num_nodes, num_nodes)` with `num_edges` nonzeros, and the node features are `(num_nodes, num_heads*head_dim)`. The edge weights have an extra `num_heads` dimension: each edge has a vector of length `num_heads`, and the feature has shape `(num_nodes, num_heads, head_dim)`. The operation is a head-wise weighted sum: `out[dst, h, d] = sum_{e in edges of dst} edge_weight[e, h] * node_feat[col_idx[e], h, d]`. We can write a custom CUDA kernel to handle this efficiently. First, let's consider how to parallelize. A straightforward approach is to assign one thread per edge and atomic-add into `out`, but atomic adds on float32 with many collisions (if destination nodes have high degree) can be slow. Since `avg_degree` is only 14 and `num_nodes` is 320, total edges is about 4480. However, we want a kernel that is fast and scale-invariant. Let's check the GPU specs.
