First, let's understand the problem. We need to implement a custom CUDA kernel for a pointwise (1x1) 2D convolution. The input shape is (batch_size, in_channels, height, width), output shape is (batch_size, out_channels, height, width). Kernel size is 1, stride 1, padding 0. This is essentially a matrix multiplication where we can reshape the input to (batch_size * height * width, in_channels) and weight to (out_channels, in_channels) and output to (batch_size * height * width, out_channels), i.e. out = X @ W^T (+ bias). A custom kernel can be more efficient than cuDNN for 1x1 conv because we can fuse bias addition and avoid cuDNN overhead, and we can use shared memory and vectorized loads.
