import torch
import torch.nn as nn
import math


class Model(nn.Module):
    """
    Rotary Position Embedding (RoPE) applied to protein residue sequences.
    Used in ESM-2 and similar protein language models.  Encodes absolute
    position via rotation of query/key pairs in 2-D subspaces, preserving
    relative-position information in dot-product attention.
    """

    def __init__(self, dim, max_seq_len=2048, base=10000.0):
        super().__init__()
        inv_freq = 1.0 / (base ** (torch.arange(0, dim, 2).float() / dim))
        self.register_buffer("inv_freq", inv_freq)
        self.max_seq_len = max_seq_len
        self.dim = dim

    def _build_cos_sin(self, seq_len: int, device: torch.device):
        t = torch.arange(seq_len, device=device, dtype=self.inv_freq.dtype)
        freqs = torch.outer(t, self.inv_freq)
        emb = torch.cat([freqs, freqs], dim=-1)
        return emb.cos(), emb.sin()

    @staticmethod
    def _rotate_half(x: torch.Tensor) -> torch.Tensor:
        x1, x2 = x.chunk(2, dim=-1)
        return torch.cat([-x2, x1], dim=-1)

    def forward(self, q: torch.Tensor, k: torch.Tensor) -> tuple:
        """
        Args:
            q: (B, H, N, D)  – queries
            k: (B, H, N, D)  – keys
        Returns:
            (q_rot, k_rot) each (B, H, N, D) with RoPE applied
        """
        N = q.shape[2]
        cos, sin = self._build_cos_sin(N, q.device)  # (N, D)
        cos = cos.unsqueeze(0).unsqueeze(0)
        sin = sin.unsqueeze(0).unsqueeze(0)
        q_rot = q * cos + self._rotate_half(q) * sin
        k_rot = k * cos + self._rotate_half(k) * sin
        return q_rot, k_rot

We need to write a custom CUDA kernel that computes Rotary Position Embedding (RoPE) applied to q and k tensors. The reference does the following: - inv_freq: shape (dim/2,) precomputed in __init__. - For sequence length N, compute t = arange(N) outer product with inv_freq => freqs shape (N, dim/2). - emb = cat([freqs, freqs], dim=-1) => shape (N, dim). - cos = emb.cos() , sin = emb.sin() => (N, dim). - Unsqueeze to (1,1,N,D). - rotate_half: split x into two halves along last dim, return [-x2, x1]. - q_rot = q * cos + rotate_half(q) * sin (broadcast over B,H). Similarly for k. Goal: Write a custom CUDA kernel that does this fused. We can precompute inv_freq on CPU and pass to kernel, but kernel is called each forward. The kernel can compute cos and sin on the fly or use precomputed buffers. Since max_seq_len=2048 and dim=64, N could be up to 2048, but typical in test is 256. We need to write a flexible kernel for any N up to max_seq_len (not necessarily fixed). The kernel should handle arbitrary B, H, N, D (with D even). Since D is small (64), N=256, B*H maybe 32. The operation is elementwise on the inner dimension but with position-dependent trig. We can compute for each element (b,h,n,d): - Determine which half of dim it is in. Since rotate_half splits into two halves: first half x1 = x[..., :D/2], second half x2 = x[..., D/2:]. - rotate_half returns [-x2, x1]. So for d in [0, D/2): rot = -x[..., d + D/2]; for d in [D/2, D): rot = x[..., d - D/2]. - cos/sin depend on n and d. Since emb = cat([freqs, freqs], dim=-1), same frequency for d and d+D/2. Specifically: - For d in [0, D/2): freq = n * inv_freq[d]. - For d in [D/2, D): freq = n * inv_freq[d - D/2]. - cos = cos(freq), sin = sin(freq). Thus q_rot[b,h,n,d] = q[b,h,n,d] * cos(n, d_mod) + rot(q)[b,h,n,d] * sin(n, d_mod). This is highly parallel. Could use a single kernel that reads q and k, writes q_rot and k_rot. Given small D (64), we might want to vectorize or use shared memory for inv_freq. Or we can just compute each thread handles one element. Since B*H*N*D may be up to 4*8*2048*64 = 4,194,304 elements, that's a lot but each thread does a few flops. Memory bound.
