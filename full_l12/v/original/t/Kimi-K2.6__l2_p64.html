import torch
import torch.nn as nn

class Model(nn.Module):
    """
    Model that performs a matrix multiplication (Gemm), followed by LogSumExp, LeakyReLU, 
    LeakyReLU, GELU, and GELU activations.
    """
    def __init__(self, in_features, out_features, bias=True):
        super(Model, self).__init__()
        self.linear = nn.Linear(in_features, out_features, bias=bias)

    def forward(self, x):
        # Gemm
        x = self.linear(x)
        # LogSumExp
        x = torch.logsumexp(x, dim=1, keepdim=True)
        # LeakyReLU
        x = torch.nn.functional.leaky_relu(x, negative_slope=0.01)
        # LeakyReLU
        x = torch.nn.functional.leaky_relu(x, negative_slope=0.01)
        # GELU
        x = torch.nn.functional.gelu(x)
        # GELU
        x = torch.nn.functional.gelu(x)
        return x

After logsumexp(dim=1, keepdim=True), it becomes [1024, 1]. So essentially, we're doing a huge matmul that produces [1024, 8192], then reducing each row with logsumexp to [1024, 1], and then applying a series of elementwise activations. The key optimization here is: 1. We don't need to materialize the full [1024, 8192] output of the linear layer. We can compute logsumexp on-the-fly. 2. However, since linear is matmul + bias, and logsumexp needs all elements per row, we can fuse the bias addition and logsumexp computation with the matmul. Actually, let me think about the math: `x @ W^T + b` where x is [B, M], W is [N, M], b is [N]. For each row i and output j: `y[i,j] = sum_k(x[i,k] * W[j,k]) + b[j]` Then `logsumexp(y, dim=1)` = `log(sum_j(exp(y[i,j])))` = `log(sum_j(exp(sum_k(x[i,k]*W[j,k]) + b[j])))` Wait, logsumexp is along dim=1 which is the column dim. So for each row i: `result[i] = log(sum_j(exp(y[i,j])))` This is a reduction over the 8192 outputs. We can compute this in a fused kernel where each block handles one row (or multiple rows) and computes the full dot products with all output features, tracks the max for numerical stability, and computes the logsumexp.
